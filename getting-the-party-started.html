paramnormal: conventionally parametermized scipy distributions.¶

Scipy has a wonderful collection of continuous and discrete probability distributions. What's especially great about them is that they all have a consitent API. So fitting a distribution to your data, or generating random numbers, or computing the CDF, it all looks pretty much the same regardless of which distribution you chose. That said, one of the costs of such a uniform API is that the names of the distribution parameters in scipy's documentation don't necessarily match what you might read about in a text book or wikipedia. I think this is best demonstrated with the lognormal distribution.

But first, some brief background¶

So let's say for a second that a random variable ($X$) is lognormally distributed if $\log(X)$ is normally distributed. Normal distributions can be specified by their mean ($\mu$) and their standard deviation ($\sigma$). This parameters are sometimes refered to as the location (of the peak of the bell curve) and scale (the width of the bell).

Visually, that looks like this when $\mu = 5.00$ and $\sigma = 2.75$:

In [2]:
from paramnormal_plots import plot_norm, format_ax

mu = 5.0
sigma = 2.75
fig = plot_norm(mu, sigma)

(the function to generate the plot is messy -- sorry about the black box)

OK, so how to make a normal distribution in scipy? It's easy. Let's stick with the parameters we defined above. We'll use scipy.stats.norm and specify it's location (loc) and scale (scale).

In [3]:
from scipy import stats
norm = stats.norm(loc=mu, scale=sigma)
print(norm)
print(norm.pdf(mu))
print(norm.cdf(mu))

<scipy.stats._distn_infrastructure.rv_frozen object at 0x10af99780>
0.145069920146
0.5

So the object is an rv_frozen object. That means that distribution's parameters have been fully specified.

Also, the PDF at $\mu$ is 0.145, which is consistent with our plot. The CDF at $\mu$ is 0.5, which tells us that have of the area is to the left of $\mu$. This makes sense as normal distributions are symmetrical around $\mu$.

We can generate random numbers with norm:

numpy.random.seed(0)

Back to lognormal distributioons¶

So now let's go back to a random variable whose logarithms are normally distributed. Let's use numpy again to generate some random numbers. The docstring of numpy.random.lognormal says:

[The] mean and standard deviation are not the values for the distribution itself, but of the underlying normal distribution it is derived from.

In other words, we're specifying the location and scale of the distribution of the logs of the data. That's a perfectly reasonable implementation, but a very important distinction to keep in mind. For simplicity, let's keep using the $\mu$ and $\sigma$ we've been using.

In [6]:
numpy.random.seed(0)
numpy.random.lognormal(mean=mu, sigma=sigma, size=(3, 3))
Out[6]:
array([[  1.89800546e+04,   4.46050566e+02,   2.18972109e+03],
       [  7.04357551e+04,   2.52298847e+04,   1.00994991e+01],
       [  2.02382213e+03,   9.78824900e+01,   1.11737029e+02]])

First, I want to point the unfortunate parameter names here. mean and sigma are not only inconsistent with numpy.random.normal, but they're also inconsistent with each other as one refers to the name of the variable and the other is the name of the symbol that represents the variable. numpy is still awesome and there's no point in bike-shedding this, though.

Moving on to scipy:

In [7]:
try:
    lognorm = stats.lognorm(loc=mu, scale=sigma)
except TypeError:
    print('Wait, what? We got a type error?')

Wait, what? We got a type error?

I've squashed the error here for neatness. But the point is that we got a type error since scipy requires a shape parameter (called s) to fully define a lognormal distribution. This is explained the Notes section of the class docstring. Let's try again, with, I dunno, s = 0:

In [8]:
lognorm = stats.lognorm(loc=mu, scale=sigma, s=0)

try:
    numpy.random.seed(0)
    lognorm.rvs(size=(3, 3))
except ValueError as err:
    print('ValueError:')
    print(err)

ValueError:
Domain error in arguments.

Again, I had to squash an error. Turns out that the parameters for scipy.lognorm are pretty confusing. I created a much more detailed notebook about them. The short version is:

  • use the shape parameter (s) to specify $\sigma$,
  • set loc to 0 (it acts as a lower bound to the distribution)
  • set scale to $\exp(\mu$)

So that's:

In [9]:
lognorm = stats.lognorm(s=sigma, loc=0, scale=numpy.exp(mu))

numpy.random.seed(0)
print('\n Scipy:')
print(lognorm.rvs(size=(3, 3)))

numpy.random.seed(0)
print('\n Numpy')
print(numpy.random.lognormal(mean=mu, sigma=sigma, size=(3, 3)))

You can also fit distributions to a dataset¶

Let's work with a lognormal distribution with known parameters: $\mu = 1.75$ and $\sigma = 0.75$

In [11]:
numpy.random.seed(0)

mu = 1.75
sigma = 0.75
fake_data = paramnormal.lognormal(mu, sigma).rvs(size=100)

Fitting the distribution is straight-forward:¶

In [12]:
params = paramnormal.fit.lognormal(fake_data)
params
Out[12]:
params(mu=1.7948494252404095, sigma=0.75590792568338261, offset=0)

You can feed those parameters directly into a distribution.

In [13]:
dist = paramnormal.lognormal(*params)

And then make a sweet plot

In [14]:
# theoretical PDF
x_hat = numpy.logspace(-0.5, 2, num=100)
y_hat = dist.pdf(x_hat)

bins = numpy.logspace(-0.5, 2, num=25)
fig, ax = pyplot.subplots()
_ = ax.hist(fake_data, bins=bins, normed=True, alpha=0.375)
ax.plot(x_hat, y_hat, zorder=2, color='g')
ax.set_xscale('log')
ax.set_xlabel('$X$')
ax.set_ylabel('Probability')
seaborn.despine()
